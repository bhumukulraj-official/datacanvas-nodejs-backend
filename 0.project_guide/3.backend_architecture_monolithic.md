# Monolithic Architecture with Microservices Readiness

## Overview

This architecture follows a monolithic approach while maintaining the flexibility to evolve into microservices when needed. It's designed for teams that want to start simple but need the ability to scale and split services later.

## Components

### 1. Load Balancer (Nginx)
- Distributes incoming traffic
- SSL termination
- Static file serving
- Rate limiting
- Basic security headers

### 2. API Gateway
- Request routing
- Authentication/Authorization
- Request validation
- Rate limiting
- API versioning

  - Version-specific implementation in separate directories
  - Future versions (v2, v3) can be added without breaking changes

- Documentation (Swagger/OpenAPI)

- Clear module boundaries
- Shared database
- Internal service communication
- Business logic isolation

### 4. Database Layer

- Connection pooling

## Rate Limiting Implementation

### Application-Level Rate Limiting
```javascript
// src/shared/middleware/rate-limit.middleware.js
const redis = require('../config/redis');
const { RateLimitError } = require('../errors');
const logger = require('../utils/logger');

// Rate limit configuration
const rateLimits = {
  // General API endpoints: 100 requests per minute
  api: {
    points: 100,
    duration: 60, // seconds
  },
  // Admin API endpoints: 1000 requests per minute
  admin: {
    points: 1000,
    duration: 60,
  },
  // WebSocket connections: 10 per second
  websocket: {
    points: 10,
    duration: 1,
  }
};

/**
 * Redis-based rate limiting middleware
 * @param {string} type Rate limit type (api, admin, websocket)
 * @returns {Function} Express middleware
 */
const rateLimiter = (type = 'api') => {
  return async (req, res, next) => {
    try {
      const ip = req.ip || req.connection.remoteAddress;
      const endpoint = req.originalUrl || req.url;
      const key = `rate_limit:${type}:${ip}`;
      
      // Get current count for this IP and endpoint
      let current = await redis.get(key);
      current = current ? parseInt(current, 10) : 0;
      
      const limit = rateLimits[type] || rateLimits.api;
      
      if (current >= limit.points) {
        // Rate limit exceeded
        logger.warn(`Rate limit exceeded: ${ip} on ${endpoint}`, { 
          ip, endpoint, limit: limit.points, current 
        });
        
        // Set headers and return error
        res.set({
          'X-RateLimit-Limit': limit.points,
          'X-RateLimit-Remaining': 0,
          'X-RateLimit-Reset': Math.floor(Date.now() / 1000) + limit.duration,
          'Retry-After': limit.duration
        });
        
        throw new RateLimitError('Rate limit exceeded');
      }
      
      // Increment counter
      await redis.multi()
        .incr(key)
        .expire(key, limit.duration)
        .exec();
      
      // Set rate limit headers
      res.set({
        'X-RateLimit-Limit': limit.points,
        'X-RateLimit-Remaining': limit.points - current - 1,
        'X-RateLimit-Reset': Math.floor(Date.now() / 1000) + limit.duration
      });
      
      next();
    } catch (error) {
      if (error.name === 'RateLimitError') {
        return res.status(429).json({
          success: false,
          error: {
            code: 'RATE_001',
            message: 'Too many requests, please try again later',
            details: { retryAfter: rateLimits[type].duration },
            timestamp: new Date().toISOString()
          }
        });
      }
      next(error);
    }
  };
};

module.exports = rateLimiter;
```

### Rate Limit Usage in Routes
```javascript
// src/api/routes/index.js
const express = require('express');
const rateLimiter = require('../../shared/middleware/rate-limit.middleware');
const authRoutes = require('../../modules/auth/routes');
const projectRoutes = require('../../modules/projects/routes');
const adminRoutes = require('./admin');

const router = express.Router();

// Public API routes with standard rate limits
router.use('/auth', rateLimiter('api'), authRoutes);
router.use('/projects', rateLimiter('api'), projectRoutes);

// Admin routes with higher rate limits
router.use('/admin', rateLimiter('admin'), adminRoutes);

module.exports = router;
```

### Nginx Configuration (Optional Additional Layer)
```nginx
# Define rate limiting zones for backup protection
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=200r/m;
limit_req_zone $binary_remote_addr zone=admin_api_limit:10m rate=2000r/m;

# API rate limiting as backup defense
location /api/v1/ {
  limit_req zone=api_limit burst=50 nodelay;
  proxy_pass http://api_gateway;
}

# Admin API rate limiting as backup defense
location /api/v1/admin/ {
  limit_req zone=admin_api_limit burst=100 nodelay;
  proxy_pass http://api_gateway;
}
```

## Conclusion

This architecture provides a balanced approach between simplicity and scalability. It's ideal for:
- Small to medium teams
- Projects with clear boundaries
- Need for quick development
- Future scalability requirements

The modular monolith approach allows for easy evolution into microservices while maintaining development speed and simplicity in the early stages.
