## Best Practices for AI-Generated Code

1. **Generate in Logical Chunks**: Follow the step-by-step approach rather than generating everything at once.

2. **Verify Each Component**: Test each module after generation before moving to the next.

3. **Check for Completeness**: AI may sometimes omit parts of the implementation. Use the documentation as a checklist.

4. **Be Specific About Structure**: Always reference the project structure when generating code to maintain consistency.

5. **Focus on API Versioning**: Remind the AI about the API versioning structure for every module implementation.

6. **Implement Security Features Early**: Ensure authentication, validation, and rate limiting are implemented early in the process.

7. **Validate Generated Code**: Ensure generated code follows the patterns and standards in the documentation.

8. **Test Thoroughly**: Use the comprehensive testing approach from the testing strategy document.

## Common Issues and Solutions

### Issue: Inconsistent API Path Prefixes

If you notice inconsistent API path prefixes (missing `/api/v1/`):

**Fix Prompt**:
```
I notice some inconsistency in the API path prefixes. Please update all routes to correctly use the /api/v1/ prefix according to our API versioning structure. Make sure the routes are organized as described in 4.development_guide.md with proper versioning.
```

### Issue: Missing Associations Between Models

If database associations are missing or incorrect:

**Fix Prompt**:
```
Please review and correct the model associations according to the database schema in 2.backend_database.md. Specifically check the associations between [MODEL1] and [MODEL2].
```

### Issue: Incomplete Error Handling

If error handling is inconsistent or incomplete:

**Fix Prompt**:
```
Please enhance the error handling in the generated code by implementing the centralized error handling approach described in the development guide. Ensure all controllers use try-catch blocks and pass errors to the central error middleware, and all error responses follow our standardized format with proper error codes.
```

### Issue: Inadequate Security Implementation

If security features are missing or incomplete:

**Fix Prompt**:
```
Please enhance the security implementation to include all required features:
1. Password policies (complexity, aging, history)
2. Account lockout (failed attempts, duration)
3. Session management (timeout, concurrent sessions)
4. Rate limiting
5. Input validation
6. CSRF protection
7. Secure headers
```

### Issue: Missing Cache Invalidation

If cache invalidation is not implemented:

**Fix Prompt**:
```
Please implement the cache invalidation strategy as described in the database schema. Ensure that updates to entities trigger appropriate cache invalidation events, and the cache service handles these events properly.
```

### Issue: Over-engineered Code

If AI generates unnecessarily complex code:

**Fix Prompt**:
```
The generated code seems more complex than needed. Please simplify it following the patterns in the development guide, which demonstrates a cleaner approach while maintaining all required functionality.
```
