# Portfolio Website Backend Development Guide

This guide provides developers with the information needed to set up, develop, and contribute to the portfolio website backend project.

## Development Environment Setup

### Prerequisites

- **Node.js:** v16+
- **PostgreSQL:** v13+
- **Redis:** v6+
- **Git**
- **Docker** (optional, for containerized development)

### Initial Setup

1. **Clone the repository:**
   ```bash
   git clone https://github.com/your-organization/portfolio-backend.git
   cd portfolio-backend
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Set up environment variables:**
   ```bash
   # Copy example environment file
   cp .env.example .env
   
   # Update variables with your local configuration
   # Edit .env file with your editor
   ```

4. **Set up the database:**
   ```bash
   # Create PostgreSQL database
   createdb portfolio_dev
   
   # Run migrations
   npm run migrate
   
   # Seed with initial data (optional)
   npm run seed
   ```

5. **Start the development server:**
   ```bash
   npm run dev
   ```

### Docker Setup (Alternative)

1. **Build and start containers:**
   ```bash
   docker-compose up -d
   ```

2. **Run migrations in container:**
   ```bash
   docker-compose exec api npm run migrate
   ```

## Coding Standards

### General Principles

1. **Modular Design**
   - Each feature should be in its own module
   - Clear separation of concerns
   - Minimize dependencies between modules

2. **Clean Code**
   - Descriptive variable and function names
   - Maximum line length: 100 characters
   - Comment complex logic
   - Follow DRY (Don't Repeat Yourself) principles

3. **Error Handling**
   - Centralized error handling
   - Consistent error responses
   - Proper error logging
   - Validation before processing

### JavaScript/TypeScript Guidelines

```javascript
// Use modern ES6+ syntax
const getUserProfile = async (userId) => {
  try {
    // Always use const/let instead of var
    const user = await userService.findById(userId);
    
    if (!user) {
      // Use custom error classes
      throw new NotFoundError('User not found');
    }
    
    // Return clean objects
    return {
      id: user.id,
      name: user.name,
      email: user.email,
      profile: user.profile
    };
  } catch (error) {
    // Proper error handling
    logger.error(`Error fetching user profile: ${error.message}`, { userId });
    throw error; // Rethrow for centralized handling
  }
};
```

## Database Guidelines

### Schema Changes

1. **Always use migrations:**
   ```bash
   # Node.js
   npm run migration:create -- create_new_table
   
   # Python
   alembic revision -m "create_new_table"
   ```

2. **Never modify production database directly**

3. **Include both up and down migrations**

4. **Test migrations before committing**

### Query Optimization

1. **Use indexes for frequently queried fields**
2. **Limit result sets when appropriate**
3. **Use pagination for large data sets**
4. **Consider query complexity and performance**

Example of a well-optimized query:

```javascript
// JavaScript with Sequelize
const getProjects = async (page = 1, limit = 10, tags = []) => {
  const offset = (page - 1) * limit;
  
  const query = {
    limit,
    offset,
    order: [['createdAt', 'DESC']],
    include: [
      {
        model: User,
        as: 'author',
        attributes: ['id', 'name', 'email']  // Select only needed fields
      }
    ]
  };
  
  // Only add where clause if tags filter is provided
  if (tags.length > 0) {
    query.where = {
      tags: {
        [Op.overlap]: tags  // Using proper PostgreSQL operator
      }
    };
  }
  
  const { rows, count } = await Project.findAndCountAll(query);
  
  return {
    projects: rows,
    pagination: {
      currentPage: page,
      totalPages: Math.ceil(count / limit),
      totalItems: count,
      itemsPerPage: limit
    }
  };
};
```

## Authentication Implementation

### JWT Setup

```javascript
// src/modules/auth/services/token.service.js
const jwt = require('jsonwebtoken');
const { v4: uuidv4 } = require('uuid');
const { TokenExpiredError, InvalidTokenError } = require('../../../shared/errors');
const config = require('../../../shared/config');
const redis = require('../../../shared/config/redis');

exports.generateTokens = async (userId) => {
  const accessToken = jwt.sign(
    { sub: userId },
    config.jwt.secret,
    { expiresIn: config.jwt.accessExpiresIn }
  );
  
  const refreshToken = uuidv4();
  const refreshExpiry = Date.now() + config.jwt.refreshExpiresIn * 1000;
  
  // Store refresh token in Redis
  await redis.set(
    `refresh_token:${refreshToken}`,
    userId,
    'EX',
    config.jwt.refreshExpiresIn
  );
  
  return {
    token: accessToken,
    refreshToken,
    expiresIn: config.jwt.accessExpiresIn,
    tokenType: 'Bearer'
  };
};

exports.verifyAccessToken = (token) => {
  try {
    const payload = jwt.verify(token, config.jwt.secret);
    return payload;
  } catch (error) {
    if (error instanceof jwt.TokenExpiredError) {
      throw new TokenExpiredError('Access token expired');
    }
    throw new InvalidTokenError('Invalid access token');
  }
};

exports.refreshToken = async (refreshToken) => {
  // Verify refresh token exists in Redis
  const userId = await redis.get(`refresh_token:${refreshToken}`);
  
  if (!userId) {
    throw new InvalidTokenError('Invalid refresh token');
  }
  
  // Delete old refresh token
  await redis.del(`refresh_token:${refreshToken}`);
  
  // Generate new token pair
  return this.generateTokens(userId);
};
```

## Caching Strategy

### Redis Caching Implementation

```javascript
// src/shared/utils/cache.js
const redis = require('../config/redis');
const logger = require('./logger');

const DEFAULT_TTL = 3600; // 1 hour in seconds

/**
 * Get data from cache
 * @param {string} key Cache key
 * @returns {Promise<any>} Cached data or null
 */
exports.get = async (key) => {
  try {
    const cachedData = await redis.get(key);
    if (cachedData) {
      return JSON.parse(cachedData);
    }
    return null;
  } catch (error) {
    logger.error(`Error getting data from cache: ${error.message}`, { key });
    return null;
  }
};

/**
 * Set data in cache
 * @param {string} key Cache key
 * @param {any} data Data to cache
 * @param {number} ttl Time to live in seconds
 */
exports.set = async (key, data, ttl = DEFAULT_TTL) => {
  try {
    await redis.set(key, JSON.stringify(data), 'EX', ttl);
  } catch (error) {
    logger.error(`Error setting data in cache: ${error.message}`, { key });
  }
};

/**
 * Delete data from cache
 * @param {string} key Cache key
 */
exports.del = async (key) => {
  try {
    await redis.del(key);
  } catch (error) {
    logger.error(`Error deleting data from cache: ${error.message}`, { key });
  }
};

/**
 * Cache wrapper for service methods
 * @param {Function} fn Service method to cache
 * @param {string} keyPrefix Cache key prefix
 * @param {number} ttl Cache TTL in seconds
 * @returns {Function} Wrapped function with caching
 */
exports.cacheWrapper = (fn, keyPrefix, ttl = DEFAULT_TTL) => {
  return async (...args) => {
    const key = `${keyPrefix}:${JSON.stringify(args)}`;
    
    // Try to get data from cache
    const cachedData = await this.get(key);
    if (cachedData) {
      return cachedData;
    }
    
    // Execute function and cache result
    const result = await fn(...args);
    await this.set(key, result, ttl);
    
    return result;
  };
};
```

## WebSocket Implementation

```javascript
// src/shared/websocket/server.js
const WebSocket = require('ws');
const http = require('http');
const url = require('url');
const { verifyAccessToken } = require('../../modules/auth/services/token.service');
const logger = require('../utils/logger');

const setupWebSocketServer = (server) => {
  const wss = new WebSocket.Server({ noServer: true });
  
  // Handle connections
  wss.on('connection', (ws, request, user) => {
    // Store user ID with connection
    ws.userId = user.sub;
    
    logger.info(`WebSocket connection established for user ${user.sub}`);
    
    // Send welcome message
    ws.send(JSON.stringify({
      type: 'status',
      payload: {
        status: 'connected',
        message: 'Connection established',
        code: 200
      },
      timestamp: new Date().toISOString(),
      messageId: Date.now().toString()
    }));
    
    // Handle messages
    ws.on('message', (message) => {
      try {
        const parsedMessage = JSON.parse(message);
        // Process message...
      } catch (error) {
        // Send error message
        ws.send(JSON.stringify({
          type: 'error',
          payload: {
            code: 'WS_003',
            message: 'Invalid message format',
            reconnect: false
          },
          timestamp: new Date().toISOString(),
          messageId: Date.now().toString()
        }));
      }
    });
    
    // Handle connection close
    ws.on('close', () => {
      logger.info(`WebSocket connection closed for user ${user.sub}`);
    });
  });
  
  // Handle upgrade requests
  server.on('upgrade', async (request, socket, head) => {
    const { pathname, query } = url.parse(request.url, true);
    
    if (pathname === '/api/v1/ws/notifications') {
      try {
        // Verify token
        const token = query.token;
        if (!token) {
          socket.destroy();
          return;
        }
        
        const user = verifyAccessToken(token);
        
        // Authenticate the connection
        wss.handleUpgrade(request, socket, head, (ws) => {
          wss.emit('connection', ws, request, user);
        });
      } catch (error) {
        logger.error(`WebSocket authentication failed: ${error.message}`);
        socket.destroy();
      }
    } else {
      socket.destroy();
    }
  });
  
  return wss;
};

module.exports = { setupWebSocketServer };
```

class AppError extends Error {
  constructor(message, statusCode) {
    super(message);

Error handling middleware:

```javascript
// src/shared/middleware/error.middleware.js
const logger = require('../utils/logger');
const { AppError } = require('../errors');

const errorHandler = (err, req, res, next) => {
  // Log all errors
  logger.error(err.message, {
    stack: err.stack,
    path: req.path,
    method: req.method
  });
  
  // If it's an operational error, send the defined response
  if (err instanceof AppError) {
    return res.status(err.statusCode).json({
      success: false,
      error: {
        code: err.code,
        message: err.message,
        details: err.details || {},
        timestamp: new Date().toISOString()
      }
    });
  }
  
  // For unknown errors, send a generic server error response
  return res.status(500).json({
    success: false,
    error: {
      code: 'SERVER_001',
      message: 'Internal server error',
      timestamp: new Date().toISOString()
    }
  });
};

module.exports = errorHandler;
```

## Pull Request Guidelines

1. **Create a feature branch from develop:**
   ```bash
   git checkout develop
   git pull
   git checkout -b feature/new-feature
   ```

2. **Make your changes and commit:**
   ```bash
   git add .
   git commit -m "feat: add new feature"
   ```

3. **Write descriptive commit messages following Conventional Commits:**
   - `feat:` - A new feature
   - `fix:` - A bug fix
   - `docs:` - Documentation changes
   - `style:` - Code style changes (formatting, etc.)
   - `refactor:` - Code changes that neither fix bugs nor add features
   - `perf:` - Performance improvements
   - `test:` - Adding or fixing tests
   - `chore:` - Changes to the build process or auxiliary tools

4. **Run tests locally before pushing:**
   ```bash
   npm test
   ```

5. **Create a pull request:**
   - Title should briefly describe the changes
   - Description should explain what was changed and why
   - Reference any related issues
   - Request reviews from relevant team members

6. **Address code review feedback**

7. **Merge only after approval and passing CI/CD checks**

## Troubleshooting Common Issues

### Database Connection Issues
- Check database connection string in .env file
- Verify that PostgreSQL service is running

- Validate JWT secret configuration
- Check token expiration settings

- Check for missing database indexes
- Review N+1 query problems
- Validate cache configuration
- Monitor resource usage
